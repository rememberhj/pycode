GitHub项目上传详细指南

从零开始学习如何将本地项目推送到GitHub仓库 - 包含详细步骤、终端命令和可视化演示

1

安装Git

Git是版本控制系统的核心工具，用于跟踪代码变更并与GitHub交互。

Git安装界面截图

验证Git安装 Terminal
$ git --version
git version 2.37.1 # 显示安装的Git版本
注意： 安装完成后，在终端输入 git --version 验证安装是否成功。
2

配置Git

设置你的用户名和邮箱，这些信息将关联到你的每一次代码提交。

设置用户信息 Terminal
$ git config --global user.name "Your Name"
$ git config --global user.email "your.email@example.com"
$ git config --list # 查看所有配置
重要： 使用你在GitHub注册时使用的邮箱地址，这样你的提交才会关联到你的GitHub账号。
3

初始化本地仓库

导航到你的项目目录并初始化Git仓库。

初始化仓库 Terminal
$ cd /path/to/your/project # 进入项目目录
$ git init
Initialized empty Git repository in /your/project/.git/
$ ls -a # 查看隐藏的.git目录
提示： 这会创建一个隐藏的 .git 目录，包含Git的所有版本控制信息。
4

添加文件到暂存区

选择要添加到版本控制的文件，准备进行提交。

添加文件 Terminal
$ git status # 查看当前状态
Untracked files: index.html, styles.css, app.js
$ git add . # 添加所有文件
$ git add index.html # 添加单个文件
说明：
  • git add . - 添加所有文件
  • git add filename - 添加特定文件
  • git add *.js - 添加所有JS文件
5

提交更改

将暂存区的文件提交到本地仓库，并添加有意义的提交信息。

提交更改 Terminal
$ git commit -m "Initial commit"
[main (root-commit) a1b2c3d] Initial commit
3 files changed, 100 insertions(+)
create mode 100644 index.html
最佳实践： 提交信息应简洁明了，说明本次提交的目的和内容变化。
6

创建GitHub仓库

在GitHub网站上创建新的远程仓库。

GitHub创建仓库界面

  1. 登录GitHub账号
  2. 点击右上角 "+" 图标
  3. 选择 "New repository"
  4. 输入仓库名称（建议与项目目录同名）
  5. 添加描述（可选）
  6. 选择公开（Public）或私有（Private）
  7. 不要初始化README、.gitignore或license文件
  8. 点击 "Create repository"
注意： 不要勾选任何初始化选项，因为我们已经有了本地仓库。
7

添加远程仓库

将GitHub仓库添加为本地仓库的远程源。

添加远程仓库 Terminal
$ git remote add origin https://github.com/username/repo-name.git
$ git remote -v # 验证远程仓库
origin https://github.com/username/repo-name.git (fetch)
origin https://github.com/username/repo-name.git (push)
说明：
  • "origin" 是远程仓库的默认名称
  • URL可以在GitHub仓库页面找到
  • 使用SSH URL（git@github.com:...）需要配置SSH密钥
8

推送到GitHub

将本地提交推送到GitHub远程仓库。

推送代码 Terminal
$ git push -u origin main
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Delta compression using up to 8 threads
Compressing objects: 100% (3/3), done.
Writing objects: 100% (5/5), 1.23 KiB | 1.23 MiB/s, done.
Total 5 (delta 0), reused 0 (delta 0)
To https://github.com/username/repo-name.git
* [new branch] main -> main
Branch 'main' set up to track remote branch 'main' from 'origin'.
说明：
  • -u 参数设置上游分支，以后可以直接使用 git push
  • 第一次推送可能需要GitHub身份验证
  • 推送成功后刷新GitHub仓库页面查看代码
